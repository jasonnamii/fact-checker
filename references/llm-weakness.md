# LLM 취약점 보정 (LLM Weakness Compensation)

LLM이 구조적으로 놓치기 쉬운 7가지 영역.

---

## 7대 취약 영역

| # | 취약점 | 보정 방법 |
|---|--------|-----------|
| 1 | **숫자 근접 혼동** (97.94 vs 98.26) | Python 산술 필수. 소수점 이하까지 비교 |
| 2 | **고유명사 스펙 환각** (시설 규모 오기) | 고유명사+숫자 조합은 반드시 검색 검증 |
| 3 | **법률·제도 일자 혼동** | 국회통과일·공포일·시행일 3개 구분 검증 |
| 4 | **지표 용어 혼동** (매출/거래액, 연결/별도) | 수치의 정확한 지표명 + 기준 확인 |
| 5 | **비율의 분모 미추적** | "무엇의 77%인가" 명시적 확인 |
| 6 | **시제·가정법 혼동** | 확정/예정/조건부 구분 태깅 |
| 7 | **attention 분산 영역** | 2패스 재스캔 (아래) |

## 2패스 재스캔 프로토콜

| 위치 | 왜 놓치는가 | 재스캔 방법 |
|------|------------|------------|
| 괄호 () 안 | 부연으로 인식, attention 저하 | 수치 포함 괄호 추출 |
| 표(table) 셀 | 셀 단위 attention 분산 | 행 단위 분해하여 셀별 추출 |
| 대시(—) 뒤 부연 | 부차적으로 처리 | 대시 구문 별도 추출 |
| HTML details/summary | 접힌 콘텐츠 스킵 경향 | details 내부 별도 추출 |

## Gotchas
- 보정 과잉 주의: "수치 포함 괄호"만 대상. 모든 괄호 추출은 노이즈
- 환각 vs 오기: 이 스킬은 인간 작성 문서의 오기를 탐지 (LLM 생성 환각과 구분)
