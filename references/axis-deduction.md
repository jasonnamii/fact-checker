# 축: 연역 검증 (Deductive Verification)

논리적 정합성을 검증한다. 소스 검색 없이 문서 내부만으로 판정 가능한 항목 + 외부 사실과의 인과 대조.

---

## 3가지 검증 유형

### 1. 내부 정합성

같은 문서 안에서 동일 팩트가 일관되게 사용되는가.

| 체크 | 방법 |
|------|------|
| 동일 수치 반복 시 일치 | grep으로 해당 수치 패턴 전수 검색. 변형(반올림·약칭) 포함 |
| 시점 표기 일관성 | "2027년" vs "2027년 1월" vs "2027년 2월" — 같은 이벤트인데 다르면 불일치 |
| 주체·대상 혼동 | "네이버"와 "네이버파이낸셜"은 다른 법인. 문서 내 혼용 체크 |

### 2. 인과 체인 검증

"A이므로 B이다"라는 주장에서 A→B 연결이 논리적으로 성립하는가.

| 패턴 | 검증 방법 |
|------|-----------|
| "X가 실패했다. 이는 Y를 증명한다" | 실패 원인이 정말 Y인지, 다른 원인은 없는지 검색 |
| "X를 가지고 있으므로 Y가 가능하다" | X→Y 연결에 빠진 전제조건이 없는지 |
| "X 시장이 열린다" | 시장 형성 조건(규제·수요·기술)이 실제로 충족되는지 |

### 3. 부재 증거 탐지 (DEEP 전용)

문서가 의도적으로 또는 무의식적으로 빠뜨린 팩트. "있어야 하는데 없는 것."

| 탐지 방법 | 예시 |
|-----------|------|
| **매출 있으면 이익은?** | 야놀자 매출 1조 기재 → 순손실 2,664억 미기재 |
| **성공 사례 있으면 실패는?** | 뮤직카우 성공 기재 → 다른 음악 STO 실패 사례 미기재 |
| **합병 기재 → 리스크는?** | 합병 일정 기재 → 공정위 불승인 가능성 미기재 |
| **경쟁자 부재** | 자사 포지션만 기재 → 동일 시장 경쟁자 미기재 |

**탐색 절차:**
1. 문서의 핵심 주장 3~5개 추출
2. 각 주장에 대해 "이 주장이 틀리려면 무엇이 사실이어야 하는가?" (프리모르템 응용)
3. 해당 반례를 WebSearch로 탐색
4. 발견 시 BLIND_SPOT으로 보고 (수정안이 아니라 "알고 있어야 할 정보"로 제시)

## Gotchas

- **인과 검증 ≠ 주장 반박**: 문서의 전략적 프레이밍을 팩트 오류로 판정하면 안 됨. "기술만으론 안 된다"는 해석이지 팩트가 아님. 인과 검증은 "전제 사실이 맞는가"에만 집중
- **부재 증거 과잉 탐지**: 모든 문서에는 빠진 정보가 있음. "의도적 누락"과 "범위 밖"을 구분. 제안서에서 경쟁사 전체 분석을 기대하는 것은 과잉
