# 축: 연역 검증 (Deductive Verification)

논리적 정합성을 검증한다. 문서 내부 일관성 + 외부 사실과의 인과 대조.

---

## 3가지 검증 유형

### 1. 내부 정합성
같은 문서 안에서 동일 팩트가 일관되게 사용되는가.
- 동일 수치 반복 시 일치 (grep 전수 검색)
- 시점 표기 일관성 ("2027년" vs "2027년 1월" vs "2027년 2월")
- 주체·대상 혼동 ("네이버"와 "네이버파이낸셜"은 다른 법인)

### 2. 인과 체인 검증
"A이므로 B이다" 주장에서 A→B 연결이 논리적으로 성립하는가.
- "X가 실패했다. 이는 Y를 증명한다" → 실패 원인이 정말 Y인지 검색
- "X를 가지고 있으므로 Y가 가능하다" → 빠진 전제조건 확인

### 3. 부재 증거 탐지 (DEEP 전용)
"있어야 하는데 없는 것." 
- 매출 있으면 이익은? 성공 있으면 실패는? 합병 있으면 리스크는?
- 프리모르템 응용: "이 주장이 틀리려면?" → 반례 WebSearch 탐색
- 발견 시 BLIND_SPOT으로 보고 (수정안이 아니라 정보 제시)

## Gotchas
- 인과 검증 ≠ 주장 반박. 전략적 프레이밍을 팩트 오류로 판정하면 안 됨
- 부재 증거 과잉 탐지: "의도적 누락"과 "범위 밖"을 구분
