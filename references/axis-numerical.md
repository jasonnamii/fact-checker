# 축: 단위·숫자 정합 (Numerical Consistency)

수치의 정확성을 Python 산술로 강제 검증한다.

---

## 검증 5유형

### 1. 산술 검증
비율 등장 시 분자/분모 직접 나눗셈. Python 필수.

### 2. 단위 변환
| 흔한 혼동 | 검증 |
|-----------|------|
| ㎡ ↔ 평 | 1평 = 3.3058㎡. "500평" ≠ "500㎡" |
| 억 ↔ 조 | 1조 = 10,000억 |
| 석(좌석) ↔ 대(차량) | 전혀 다른 단위 |
| % ↔ %p | "56%에서 60%로" = 4%p 상승 ≠ 4% 상승 |

### 3. 합산 기준
- 부문합산 vs 연결매출 (내부거래 제거)
- 매출 vs 거래액(TTV/GMV)
- GAAP vs Non-GAAP

### 4. 시계열 검증
- 연도 혼동, 분기 vs 연간, 예정 vs 확정

### 5. 반올림·근사
→ references/tolerance.md 참조

## Gotchas
- Python 미실행 = FAIL. 근접 수치(97.94 vs 98.26)가 가장 위험
- 검색 결과의 수치도 산술 재검증 필요
- 합산 함정: 부문별 합계를 "매출"로 쓰면 연결매출과 불일치
