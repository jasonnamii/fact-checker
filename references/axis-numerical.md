# 축: 단위·숫자 정합 (Numerical Consistency)

수치의 정확성을 Python 산술로 강제 검증한다.

---

## 검증 5유형

### 1. 산술 검증

문서가 비율을 제시하면, 분자/분모로 직접 나눗셈하여 일치 여부 확인.

```python
# 예: "거래수수료가 매출의 97.94%"
numerator = 15307  # 거래수수료 매출 (억)
denominator = 15578  # 전체 매출 (억)
actual = numerator / denominator * 100
claimed = 97.94
print(f"실제: {actual:.2f}% vs 문서: {claimed}% → 차이: {abs(actual-claimed):.2f}%p")
```

**규칙:** 비율·백분율이 등장하면 무조건 Python 검증. "검색 결과에서 가져온 수치"도 산술로 재검증.

### 2. 단위 변환

| 흔한 혼동 | 검증 |
|-----------|------|
| ㎡ ↔ 평 | 1평 = 3.3058㎡. "500평" ≠ "500㎡" (500㎡ ≈ 151평) |
| 억 ↔ 조 | 1조 = 10,000억. "1조 763억" = "10,763억" |
| 석(좌석) ↔ 대(차량) | 전혀 다른 단위. 문맥에서 혼동 체크 |
| % ↔ %p | "56%에서 60%로 올랐다" = 4%p 상승 ≠ 4% 상승 |

### 3. 합산 기준

| 체크 | 방법 |
|------|------|
| 부문합산 vs 연결매출 | CP+ES 합산 ≠ K-IFRS 연결매출 (내부거래 제거). 공시 기준 확인 |
| 매출 vs 거래액(TTV/GMV) | 플랫폼 기업에서 가장 흔한 혼동. 거래액은 매출의 10~100배일 수 있음 |
| GAAP vs Non-GAAP | 조정 EBITDA와 영업이익은 다름 |

### 4. 시계열 검증

| 체크 | 방법 |
|------|------|
| 연도 혼동 | "2024년 수치"를 "2025년"으로 기재하지 않았는가 |
| 분기 vs 연간 | Q4 수치를 연간으로 혼동 |
| 예정 vs 확정 | "2026년 9월 완료 예정"과 "2026년 9월 완료"는 다름 |

### 5. 반올림·근사

반올림 허용 여부는 문서 용도에 따라 판정 → references/tolerance.md 참조

## Gotchas

- **Python 미실행 = FAIL**: "대략 맞다"는 판정은 이 축에서 금지. 근접 수치(97.94 vs 98.26)가 가장 위험
- **검색 결과의 수치도 재검증**: WebSearch로 얻은 "98.26%"도 분자/분모 확인 필요. 보도가 계산 실수할 수 있음
- **합산 함정**: 부문별 합계를 "매출"로 쓰면 검증자가 연결매출과 대조 시 불일치 발생. 기준 명시 필수
